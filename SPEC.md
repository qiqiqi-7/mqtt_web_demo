# MQTT Web 调试工具 — SPEC.md

## 1. 概念与愿景

**MQTT Broker Manage** — 面向 IoT 开发者的专业网页端调试工具。堪称 "MQTT 界的 Postman"——简洁、高效、信息密度高，在提供完整 MQTT 流量可视化的同时不干扰你的工作。

**产品调性：** 开发者工具美学。深色主题，高对比度文字，终端风格的颜色点缀。基于 React + TypeScript 构建。

## 4. 功能与交互

### F1: 连接管理

#### 4.1.1 添加连接（弹窗）

**基础配置**
| 字段 | 类型 | 必填 | 默认值 |
|------|------|------|--------|
| 连接名称 | 文本 | 是 | 自动从 URL 生成 |
| 协议版本 | 下拉选择 | 是 | MQTT 5.1 |
| 协议 | 下拉选择 | 是 | ws:// |
| Broker 地址 | 文本 | 是 | — |
| 端口 | 数字 | 是 | 8084 (wss) / 8083 (ws) |
| Client ID | 文本 | 否 | 自动生成 UUID |

**高级配置**
| 字段 | 类型 | 必填 | 默认值 |
|------|------|------|--------|
| 认证方式 | 下拉选择 | 否 | 无 |
| 用户名 | 文本 | 否 | — |
| 密码 | 密码 | 否 | — |
| TLS CA 证书 | 文件上传 | 否 | — |
| Keepalive | 数字 | 否 | 60秒 |
| 自动重连 | 复选框 | 否 | true |
| Clean Session | 复选框 | 否 | true |
| Last Will | 折叠区域 | 否 | 折叠 |

**Last Will 区域：**
| 字段 | 类型 | 必填 | 默认值 |
|------|------|------|--------|
| 遗嘱主题 | 文本 | 否 | — |
| 遗嘱 QoS | 下拉选择 | 否 | 0 |
| 遗嘱 Retained | 复选框 | 否 | false |
| 遗嘱载荷 | 文本域 | 否 | — |

**认证方式选项：**
- 无（None）
- 用户名 + 密码（Username + Password）
- TLS CA 证书（TLS Certificate）

**端口默认值：**
- ws:// → 8083
- wss:// → 8084

#### 4.1.2 连接状态
| 状态 | 视觉表现 |
|------|---------|
| 已连接（Connected） | 绿色圆点 + "已连接" |
| 未连接（Disconnected） | 灰色圆点 + "未连接" |
| 连接中（Connecting） | 黄色脉冲圆点 + "连接中..." |
| 重连中（Reconnecting） | 黄色脉冲圆点 + "重连中..." |
| 错误（Error） | 红色圆点 + 错误提示 |

#### 4.1.3 连接切换
- 点击连接卡片 → 选中该连接
- 如果已有其他连接处于连接状态 → 先断开 → 再连接新连接
- 连接列表高亮显示当前激活的连接（卡片带蓝色左边框）
- 切换连接后，订阅列表自动更新为新连接的订阅

#### 4.1.4 编辑连接
- 右键连接 → 上下文菜单 → 编辑
- 弹出与"添加连接"相同的弹窗，预填充当前值
- 密码字段显示当前密码的掩码（`••••••`）

#### 4.1.5 删除连接
- 右键连接 → 删除
- 确认弹窗："确定删除 [name]？"
- 删除连接及其所有订阅
- 如果在连接状态下删除 → 先断开再删除

#### 4.1.6 密码存储
- 密码使用 Base64 + 自定义密钥加密（简单混淆）
- 仅在用户保存连接时存储
- 加载时解密显示（仍为掩码形式）

### F2: 订阅管理

#### 4.2.1 添加订阅（弹窗）
| 字段 | 类型 | 必填 | 默认值 |
|------|------|------|--------|
| 主题 pattern | 文本 | 是 | — |
| QoS | 下拉选择 | 是 | 0 |

- 主题支持 MQTT 通配符：`#`（多级）、`+`（单级）
- 示例：`devices/+/temperature`、`sensor/#`

#### 4.2.2 订阅展示
- 列表显示在第二栏（当前连接下）
- 每项显示：
  - 主题 pattern
  - QoS 徽章（彩色药丸状）
  - 删除按钮（悬停显示）

#### 4.2.3 取消订阅
- 点击删除按钮 → 即时移除
- 无需确认（低风险操作）

### F3: 消息流

#### 4.3.1 消息展示
- 列：时间戳 | 方向 | 主题 | QoS | 载荷预览
- 方向：
  - ↓ 入站（蓝色箭头）
  - ↑ 出站（绿色箭头）
- QoS 徽章：
  - QoS 0：灰色徽章
  - QoS 1：蓝色徽章
  - QoS 2：紫色徽章
- 载荷预览：前 50 字符，"..." 截断
- Retained 指示器：★ 图标（黄色），位于保留消息左侧
- 保留消息带淡黄色背景

#### 4.3.2 消息展开
- 点击消息行 → 在下方展开
- 显示完整载荷（格式化/美化视图）
- JSON 检测 → 语法高亮
- 复制按钮
- 再次点击收起

#### 4.3.3 消息筛选
| 筛选器 | 说明 |
|--------|------|
| 搜索输入框 | 按主题或载荷内容筛选（支持正则） |
| 方向筛选 | 全部 / 入站 / 出站 |
| 主题筛选 | 以下拉菜单选择订阅的主题 |

- "清除筛选"按钮重置所有筛选器

#### 4.3.4 消息操作
- 悬停消息 → 显示：复制主题图标、复制载荷图标、删除图标
- 复制成功显示短暂"已复制！"提示

#### 4.3.5 消息计数
- 消息条数无限制
- 新消息追加到底部
- 新消息到达时自动滚动到底部（除非用户已向上滚动）

### F4: 消息发布

#### 4.4.1 主题输入
- 文本输入框
- 校验：不能为空，不能包含通配符
- 无效时显示错误提示

#### 4.4.2 载荷编辑器
- 多行文本域
- 字符计数显示
- 自动检测 JSON：
  - 如果是合法 JSON → 显示 "JSON" 徽章
  - 点击徽章 → 切换原始/格式化视图
- 手动美化按钮：将 JSON 按 2 空格缩进格式化

#### 4.4.3 发布选项
| 选项 | 类型 | 默认值 |
|------|------|--------|
| QoS | 下拉选择 | 0 |
| Retained | 复选框 | false |

#### 4.4.4 发布操作
- 发送按钮
- 以下情况禁用：无连接、无主题、无载荷
- 成功时：按钮短暂闪绿
- 失败时：闪红 + 错误提示弹窗

#### 4.4.5 发布配置（保存）
- 将当前 topic+payload+qos+retained 保存为预设
- 为预设命名
- 加载预设：填充表单为保存的值
- 管理预设：列出已保存预设，删除不需要的
- 预设按连接保存

### F5: 导入/导出
- **不支持**（按需求）

## 5. 组件清单

### C1: ConnectionCard（连接卡片）
- **默认态**：深色卡片，连接名称、协议徽章、状态圆点
- **悬停态**：边框变亮、阴影
- **选中态（激活中）**：蓝色左边框（3px）、背景亮度增加
- **已连接**：绿色状态圆点，名称带绿色色调
- **未连接**：灰色状态圆点

### C2: SubscriptionItem（订阅项）
- **默认态**：行，含主题文本、QoS 徽章
- **悬停态**：背景高亮，显示删除图标
- **有新消息时**：加粗文本（可选徽章）

### C3: MessageRow（消息行）
- **默认态**：单行，斑马纹背景（微妙）
- **悬停态**：背景高亮，显示操作图标
- **展开态**：行下方展开，显示完整载荷和语法高亮
- **入站**：蓝色箭头（↓）
- **出站**：绿色箭头（↑）
- **保留消息**：黄色左边框，★ 图标

### C4: QoS Badge（QoS 徽章）
- **QoS 0**：灰色背景 `#5a5a5a`，白色文字
- **QoS 1**：蓝色背景 `#569cd6`，白色文字
- **QoS 2**：紫色背景 `#c586c0`，白色文字

### C5: TopicInput（主题输入框）
- **默认态**：深色输入框，等宽字体
- **聚焦态**：蓝色边框发光
- **错误态**：红色边框，下方显示错误信息

### C6: PayloadEditor（载荷编辑器）
- **默认态**：深色文本域
- **聚焦态**：蓝色边框发光
- **JSON 模式**：应用语法高亮

### C7: Modal（弹窗）
- **遮罩层**：`rgba(0,0,0,0.6)` 半透明黑色
- **容器**：`#252526` 背景，520px 宽度，居中
- **标题栏**：标题 + 关闭 X 按钮
- **内容区**：表单字段，16px 间距
- **底部栏**：取消 + 保存按钮，右对齐
- **入场动画**：淡入 + 从 0.95 缩放（150ms）

### C8: Toast Notification（提示弹窗）
- **位置**：右下角，距边缘 20px
- **成功**：绿色左边框
- **错误**：红色左边框
- **信息**：蓝色左边框
- **持续时间**：默认 4 秒，错误 8 秒

### C9: EmptyState（空状态）
- **无连接**："请先添加一个连接" + 图标
- **无订阅**："当前连接暂无订阅" + 提示文字

### 数据模型

**Connection（连接）**
```javascript
{
  id: string,                // UUID
  name: string,              // 连接名称
  protocolVersion: '3.1.1' | '5.1',  // MQTT 协议版本
  protocol: 'ws' | 'wss',   // 协议
  host: string,              // Broker 地址
  port: number,              // 端口
  clientId: string,          // 客户端 ID（留空自动生成）
  auth: 'none' | 'password' | 'tls',  // 认证方式
  username: string | null,
  password: string | null,   // 加密存储
  tlsCaCert: string | null,  // Base64 编码的 CA 证书
  keepalive: number,         // 秒
  autoReconnect: boolean,
  cleanSession: boolean,
  lastWill: {
    topic: string | null,
    qos: 0 | 1 | 2,
    retained: boolean,
    payload: string | null
  } | null
}
```

**Subscription（订阅）**
```javascript
{
  id: string,                // UUID
  connectionId: string,      // 父连接 UUID
  topic: string,             // 主题 pattern
  qos: 0 | 1 | 2
}
```

**Message（消息）**
```javascript
{
  id: string,                // UUID
  connectionId: string,      // 所属连接
  direction: 'in' | 'out',  // 方向
  topic: string,
  qos: 0 | 1 | 2,
  retained: boolean,
  payload: string,
  timestamp: number          // Unix 时间戳（毫秒）
}
```

**PublishPreset（发布预设）**
```javascript
{
  id: string,
  connectionId: string,
  name: string,
  topic: string,
  payload: string,
  qos: 0 | 1 | 2,
  retained: boolean
}
```

### 数据模型

**Connection（连接）**
```typescript
interface Connection {
  id: string;
  name: string;
  protocolVersion: '3.1.1' | '5.1';
  protocol: 'ws' | 'wss';
  host: string;
  port: number;
  clientId: string;
  auth: 'none' | 'password' | 'tls';
  username: string | null;
  password: string | null;  // 加密存储
  tlsCaCert: string | null;
  keepalive: number;
  autoReconnect: boolean;
  cleanSession: boolean;
  lastWill: LastWill | null;
}

interface LastWill {
  topic: string | null;
  qos: 0 | 1 | 2;
  retained: boolean;
  payload: string | null;
}
```

**Subscription（订阅）**
```typescript
interface Subscription {
  id: string;
  connectionId: string;
  topic: string;
  qos: 0 | 1 | 2;
}
```

**Message（消息）**
```typescript
interface Message {
  id: string;
  connectionId: string;
  direction: 'in' | 'out';
  topic: string;
  qos: 0 | 1 | 2;
  retained: boolean;
  payload: string;
  timestamp: number;
}
```

**PublishPreset（发布预设）**
```typescript
interface PublishPreset {
  id: string;
  connectionId: string;
  name: string;
  topic: string;
  payload: string;
  qos: 0 | 1 | 2;
  retained: boolean;
}
```

## 7. 验收标准

### AC1: 连接管理
- [ ] 用户可添加包含基础+高级配置的连接
- [ ] 用户可编辑已有连接
- [ ] 用户可删除连接（带确认）
- [ ] 任意时刻只能有一个连接处于激活状态
- [ ] 切换连接时自动断开上一个、连接新的
- [ ] 密码加密存储于 LocalStorage
- [ ] 可上传 TLS CA 证书

### AC2: 订阅管理
- [ ] 用户可订阅主题（支持通配符）

- [ ] 订阅列表仅显示当前连接的订阅
- [ ] QoS 设置生效

- [ ] 搜索和筛选功能正常

- [ ] JSON 美化功能正常

### AC5: 持久化
- [ ] 连接配置刷新后不丢失

- [ ] 发布预设刷新后不丢失
