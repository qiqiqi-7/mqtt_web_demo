# MQTT Web Demo - Claude Code 工作规范

## 开发流程规范

### 1. 需求处理流程

**遇到需求或问题时，优先给出解决方案，而不是动手写代码。**

具体步骤：
1. **理解需求** - 确认用户想要什么
2. **分析问题** - 找出问题的根本原因
3. **给出方案** - 提供2-3个可选方案，附带优缺点对比
4. **等待确认** - 方案确认后再进行编码
5. **实现验证** - 编码后进行测试验证

### 2. 编码前规范

在动手写代码之前，必须先：
1. 理解完整的需求
2. 设计好代码结构或布局（如有必要，提供草图）
3. 考虑边界情况和错误处理
4. 确认实现方案后再开始编码

### 3. 调试和问题排查

遇到问题时：
1. **先分析** - 不要直接猜测原因
2. **给出可能的原因列表**
3. **提供解决方案**
4. **如果需要编码，提供修复方案并等待确认**

### 4. 示例

#### 场景1：用户报告布局问题

**错误做法**：直接修改代码，尝试各种方案碰运气

**正确做法**：
> 方案1：调整 padding 和 margin
> 方案2：改用 flexbox 布局
> 方案3：重写布局组件
>
> 推荐方案1，因为改动最小，对现有代码影响最小。
> 需要我实现吗？

#### 场景2：用户报告功能不工作

**错误做法**：猜测是某个变量问题，直接修改试试

**正确做法**：
> 可能的原因：
> 1. 事件没有正确绑定
> 2. 状态更新后没有触发重新渲染
> 3. 回调函数被重复覆盖
>
> 需要我进一步排查哪个原因？或者提供您的操作步骤，我来分析最可能的原因。

## 项目技术栈

- React 18 + TypeScript
- Vite
- MQTT.js v5 (WebSocket 连接)
- Font Awesome 6 (图标)
- react-virtuoso (虚拟滚动，支持动态高度)
- CSS Variables (主题管理)

```
src/

├── services/       # MQTT 服务层
```
